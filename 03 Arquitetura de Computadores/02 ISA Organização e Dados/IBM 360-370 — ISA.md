# IBM 360/370 — ISA

## Definição

[[ISA]] [[RISC e CISC|CISC]] de 32 bits criada pela IBM em 1964. Primeira arquitetura a definir explicitamente o conceito de **família de computadores** com código compatível. O termo "arquitetura de computador" foi cunhado pela IBM para descrever a IBM 360.

## Relações (SPO)

- IBM 360 → cunhou o conceito de → arquitetura de computador (ISA unificada entre modelos)
- IBM 360 → introduziu → byte de 8 bits, endereçamento de byte, dados de 8/16/32/64 bits
- IBM 370 → adicionou a → 360: memória virtual (1972), TLB
- IBM 360 → influenciou → todas as ISAs subsequentes com espaço de endereçamento uniforme
- IBM 360/370 → representa → linha [[RISC e CISC|CISC]] com ênfase em mainframes e compatibilidade binária

Ver história em [[História da Computação#IBM System/360 (1964)]].

## Inovações Arquiteturais da IBM 360 (1964)

1. **Arquitetura de 32 bits** — registradores e dados de 32 bits
2. **Memória endereçável por byte** com bytes de 8 bits (padrão hoje universal)
3. **Múltiplos tamanhos de dados** — 8, 16, 32 e 64 bits
4. **Ponto flutuante** — single precision (32 bits) e double precision (64 bits)
5. **Família de computadores** — 6 modelos iniciais com razão de desempenho 25:1, todos rodando o mesmo software

## Evolução IBM 370 (1971)

- Extensões significativas descritas por Case e Padegs [1978]
- **Memória virtual** (modelos com VM em 1972) — aqui o termo *translation lookaside buffer* (TLB) foi cunhado
- 1984: endereçamento de 24 bits esgotado → 370-XA (eXtended Architecture) com 31 bits de endereço

> [!warning] Endereçamento 24 bits → problema histórico
> A IBM 360 usava 24 dos 32 bits de endereço. Programadores armazenaram dados nos 8 bits "não usados", impedindo expansão. Lição: bits reservados devem ser verificados e causar trap se modificados.

## Registradores

- **16 registradores de propósito geral** de 32 bits (R0–R15)
- **Registradores de ponto flutuante** separados
- **PSW (Program Status Word)** — modo da máquina, condições, nível de interrupção

## Formatos de Instrução

| Formato | Comprimento | Descrição |
|---|---|---|
| **RR** | 2 bytes | Register-Register |
| **RX** | 4 bytes | Register + displacement(base) |
| **RS** | 4 bytes | Register + register + displacement |
| **SI** | 4 bytes | Immediate + storage address |
| **SS** | 6 bytes | Storage-Storage (operações de string/decimal) |

Instruções de comprimento variável (2, 4 ou 6 bytes) — típico de [[RISC e CISC|CISC]].

## Frequência de Uso (Medições)

Distribuição de execução para 4 programas reais (PL/I compiler, FORTRAN, COBOL runtimes):

| Categoria | Média |
|---|---|
| Controle (BC, BAL) | ~25–30% |
| Data transfer (L, LR, ST, MVI) | ~25–35% |
| Aritmético/lógico (A, AR, SR, LA) | ~15–25% |
| FP | ~7% |
| Decimal/string (MVC, AP, ZAP) | ~3–20% (COBOL: muito mais) |

> [!info] Impacto nos ISAs RISC
> Medições como estas (feitas por Shustek [1978] e pelo grupo de Toronto [1975]) revelaram que poucas instruções respondem por quase toda a execução — motivação direta para o RISC: se 20% das instruções aparecem em 80% do tempo, simplificar essas 20% e fazer hardware otimizado para elas.

## Herança

- Sistema/370 (1971) → 4300, 3080, 3090, 390, z/Architecture (ainda ativa)
- Cada geração manteve compatibilidade binária com o System/360 original
- TLB (IBM 370, 1972) → padrão universal em todas as CPUs com [[Memória Virtual]]
- Termo "arquitetura de computador" — Amdahl, Blaauw e Brooks [1964] para a IBM 360
