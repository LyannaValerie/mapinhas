# Documento de Especificação de Software

%% criado pelo Codex %%

## Definição

Documento que estabelece regras entre comportamento desejado, interfaces e restrições do software. Em termos práticos, especificação é desenho de interface com restrições verificáveis.

## Escopo

A especificação não precisa ser escrita apenas antes da implementação. Ela pode ser preparada antes, durante ou depois, desde que capture regras úteis para validar e manter o sistema.

## Relações

- Refina o [[Documento de Requisitos de Software]].
- Define contratos consumidos por implementação, teste e manutenção.
- Em sistemas de baixo nível, aproxima documentação do software da documentação oficial de hardware.
