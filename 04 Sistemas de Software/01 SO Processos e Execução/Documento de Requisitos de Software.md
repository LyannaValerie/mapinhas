# Documento de Requisitos de Software

%% criado pelo Codex %%

## Definição

Documento que registra os efeitos que a máquina deve produzir no [[Documentos de Domínio|domínio do problema]] por meio de sua programação.

## Escopo

Requisito não é desenho de tela, algoritmo, estrutura de dados ou plano de implementação. O foco é o resultado observável no domínio, não a técnica usada dentro do software.

## Relações

- Precede ou orienta o [[Documento de Especificação de Software]].
- Depende de entendimento explícito do [[Documentos de Domínio|domínio do problema]].
- Ajuda a separar comportamento esperado de detalhes de [[Sistema de Compilação|implementação]].
