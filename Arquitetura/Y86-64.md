# Y86-64

ISA pedagógica derivada do [[Intel x86|x86-64]] por simplificação. Usada em CS:APP (Bryant & O'Hallaron, Cap. 4) para ensinar projeto de processadores. Mantém os conceitos centrais; remove complexidades do x86-64 real.

## Relações (SPO)
- Y86-64 → subconjunto simplificado de → x86-64
- Y86-64 → possui → 15 registradores de 64 bits
- Y86-64 → usa → endereçamento absoluto (não PC-relativo)
- Y86-64 → instruções de → 1 a 10 bytes
- Y86-64 → híbrido entre → [[RISC e CISC|RISC e CISC]]

## Estado Visível ao Programador

| Componente | Descrição |
|---|---|
| **RF** | 15 registradores de 64 bits: `%rax`–`%r14` (sem `%r15`) |
| **PC** | Contador de programa |
| **CC** | 3 códigos de condição: ZF, SF, OF |
| **Stat** | Código de status da execução |
| **MEM** | Memória (array de bytes, endereçamento virtual) |

### Registradores
IDs 0x0–0xE; ID **0xF = sem registrador** (usado em encodings quando operando não existe).

| ID | Registrador | ID | Registrador |
|---|---|---|---|
| 0 | %rax | 8 | %r8 |
| 1 | %rcx | 9 | %r9 |
| 2 | %rdx | A | %r10 |
| 3 | %rbx | B | %r11 |
| 4 | %rsp | C | %r12 |
| 5 | %rbp | D | %r13 |
| 6 | %rsi | E | %r14 |
| 7 | %rdi | F | (nenhum) |

`%rsp` é o stack pointer usado por push/pop/call/ret.

### Códigos de Condição
Ajustados pelas 4 operações inteiras (OPq):

| Flag | Significado |
|---|---|
| **ZF** | resultado zero |
| **SF** | resultado negativo (sign) |
| **OF** | overflow com sinal |

### Códigos de Status (Stat)

| Valor | Nome | Significado |
|---|---|---|
| 1 | AOK | Operação normal |
| 2 | HLT | Instrução `halt` executada |
| 3 | ADR | Endereço inválido (instrução ou dado) |
| 4 | INS | Instrução inválida |

Qualquer exceção (HLT, ADR, INS) para o processador.

## Conjunto de Instruções

### Encoding: byte de instrução
```
[ icode (4 bits) | ifun (4 bits) ]
```
`icode` = código da instrução; `ifun` = função específica (operação, condição de desvio).

### Tabela de instruções

| Instrução | icode | ifun | Bytes | Descrição |
|---|---|---|---|---|
| `halt` | 0 | 0 | 1 | Para execução |
| `nop` | 1 | 0 | 1 | Nenhuma operação |
| `rrmovq rA, rB` | 2 | 0 | 2 | rB ← rA |
| `cmovXX rA, rB` | 2 | 1–6 | 2 | rB ← rA se condição |
| `irmovq V, rB` | 3 | F | 10 | rB ← V (imediato) |
| `rmmovq rA, D(rB)` | 4 | 0 | 10 | Mem[rB+D] ← rA |
| `mrmovq D(rB), rA` | 5 | 0 | 10 | rA ← Mem[rB+D] |
| `OPq rA, rB` | 6 | 0–3 | 2 | operação inteira |
| `jXX Dest` | 7 | 0–6 | 9 | desvio condicional |
| `call Dest` | 8 | 0 | 9 | push PC+9; PC ← Dest |
| `ret` | 9 | 0 | 1 | pop; PC ← topo |
| `pushq rA` | A | 0 | 2 | %rsp−=8; Mem[%rsp] ← rA |
| `popq rA` | B | 0 | 2 | rA ← Mem[%rsp]; %rsp+=8 |

### Operações inteiras (OPq, icode=6)

| ifun | Instrução | Operação |
|---|---|---|
| 0 | `addq rA, rB` | rB ← rB + rA |
| 1 | `subq rA, rB` | rB ← rB − rA |
| 2 | `andq rA, rB` | rB ← rB & rA |
| 3 | `xorq rA, rB` | rB ← rB ^ rA |

Todas ajustam ZF, SF, OF.

### Desvios (jXX, icode=7) e Movimentos Condicionais (cmovXX, icode=2)

| ifun | Sufixo | Condição |
|---|---|---|
| 0 | (nenhum / jmp) | sempre |
| 1 | le | ≤ com sinal |
| 2 | l | < com sinal |
| 3 | e | igual (ZF=1) |
| 4 | ne | diferente |
| 5 | ge | ≥ com sinal |
| 6 | g | > com sinal |

## Encoding de Instruções

### Estrutura geral
```
Byte 0:   [ icode | ifun ]
Byte 1:   [ rA    | rB   ]   (se necessário; rF=0xF=ausente)
Bytes 2–9: constante V ou deslocamento D (little-endian, 8 bytes)
```

Diferenças de x86-64:
- Registradores sempre em posição fixa (byte 1)
- Constante sempre 8 bytes (nunca 1/2/4)
- Desvios usam endereço absoluto, não PC-relativo
- Resultado: instruções menos compactas, decodificação mais simples

### Exemplo de encoding
`rmmovq %rsp, 0x123456789abcd(%rdx)`:
- `40` (rmmovq, icode=4, ifun=0)
- `42` (rA=%rsp=4, rB=%rdx=2)
- `cd ab 89 67 45 23 01 00` (deslocamento em little-endian)
→ `4042cdab896745230100`

## Características RISC/CISC

| CISC (herdado do x86) | RISC (adotado) |
|---|---|
| Códigos de condição | Arquitetura load/store |
| Instruções de comprimento variável | Encoding regular de registradores |
| Pilha para endereços de retorno | Argumentos em registradores |

## Convenções Especiais

### `pushq %rsp`
Empurra o valor **original** de `%rsp` (antes do decremento) — mesma convenção do x86-64.

### `popq %rsp`
Carrega em `%rsp` o valor **lido da memória** (ignora o incremento) — equivale a `mrmovq (%rsp), %rsp`.
